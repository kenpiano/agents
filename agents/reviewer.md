# Reviewer Agent Group

## Overview
The Reviewer group specializes in evaluating and providing feedback on all types of deliverables. This group contains four specialized sub-agents that ensure quality through thorough review.

## Sub-Agents

| Sub-Agent | File | Specialization |
|-----------|------|----------------|
| Plan Reviewer | [plan-reviewer.md](./reviewer/plan-reviewer.md) | Project plans, task breakdowns, strategy |
| Design Reviewer | [design-reviewer.md](./reviewer/design-reviewer.md) | System, code, and document designs |
| Document Reviewer | [document-reviewer.md](./reviewer/document-reviewer.md) | Documentation quality and accuracy |
| Code Reviewer | [code-reviewer.md](./reviewer/code-reviewer.md) | Code quality, correctness, maintainability |

## When to Use

- **Plan Reviewer**: When reviewing project plans, strategies, or task breakdowns
- **Design Reviewer**: When reviewing system architecture, code design, or document design
- **Document Reviewer**: When reviewing documentation for accuracy and completeness
- **Code Reviewer**: When reviewing code changes for quality and correctness

## Collaboration Points

### Receives Input From
- **Manager**: Review requests
- **Planner**: Plans for review
- **Designer**: Designs for review
- **Editor**: Code/docs for review

### Provides Output To
- **Manager**: Review results and recommendations
- **Editor**: Feedback requiring changes
- **Quality Checker**: Review findings
- **Planner**: Issues requiring re-planning

## Review Best Practices

1. **Be Constructive**: Provide actionable feedback
2. **Be Specific**: Point to exact locations/issues
3. **Be Balanced**: Acknowledge good work too
4. **Be Timely**: Complete reviews promptly
5. **Be Thorough**: Don't rush through reviews
6. **Be Respectful**: Critique the work, not the person
7. **Prioritize Issues**: Distinguish critical from minor
8. **Suggest Solutions**: Offer alternatives when possible
